## Architecture Summary

### Backend Stack
- **Framework:** Express.js (TypeScript)
- **Authentication:** Session cookies + bcrypt hashing
- **ML Bridge:** Node.js ↔ Python via base64-encoded data
- **Storage:** File-based (JSON) with in-memory Maps
- **Concurrency Control:** p-limit (max 5 parallel processes)

### Machine Learning Stack
- **Models:** XGBoost + Random Forest ensemble
- **Feature Engineering:** 56 features (6 base + 50 TF-IDF)
- **Vectorization:** TfidfVectorizer (max_features=50, ngram_range=(1,2))
- **Validation:** 80/20 stratified train/test split
- **Performance:** ~91% accuracy on test set

### Frontend Stack
- **Framework:** React 18 with TypeScript
- **State Management:** TanStack Query
- **Form Validation:** Zod schemas
- **Security:** Custom SafeHtml/SafeText components for XSS prevention
- **Reusability:** Custom hooks for mutations, filtering, and formatting

## Known Limitations

### Current
1. File-based storage (scales to ~10,000 vulnerabilities)
2. Single-threaded Python ML model
3. No distributed caching
4. No API rate limiting (recommended for production)

### Mitigations
- Archive old data periodically
- Use concurrent process limiting (now implemented)
- Consider Redis for production deployments
- Add rate limiting middleware per environment
